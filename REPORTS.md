# Test Reports Guide

This document explains how to access and use the test reports generated by the test suite.

## Generated Reports

After running `pytest`, the following reports are automatically generated in the `reports/` directory:

### 1. **HTML Report** (`reports/report.html`)
- **Location**: `reports/report.html`
- **Purpose**: Comprehensive HTML report with test results, pass/fail status, and test metadata
- **How to access**: 
  - Open the file in a web browser: `reports/report.html`
  - Shows detailed information about each test execution
  - Includes test duration, platform info, and Python version

### 2. **Allure Reports** (`reports/allure-results/`)
- **Location**: `reports/allure-results/`
- **Purpose**: Generates raw Allure data for creating interactive dashboards
- **Files generated**:
  - `*.json` files containing test results
  - `*-attachment.txt` files containing test logs and console output
  - `*-container.json` files for test hierarchy

#### How to View Allure Report
```bash
# Install allure CLI (if not already installed)
npm install -g allure-commandline

# Generate and serve the Allure HTML report
allure serve reports/allure-results
```

This will open an interactive dashboard showing:
- Test statistics and trends
- Individual test details
- Timeline of test execution
- Failed tests with attachments

### 3. **Logs** (`reports/logs/pytest.log`)
- **Location**: `reports/logs/pytest.log`
- **Purpose**: Complete test execution logs with timestamps
- **Content includes**:
  - WebDriver start/stop messages
  - Navigation logs
  - Test execution flow
  - Any errors or warnings during test execution
- **View**: Open with any text editor

### 4. **Screenshots** (`reports/screenshots/`)
- **Location**: `reports/screenshots/`
- **Purpose**: Automatically captures screenshots when tests fail
- **Files**: Named with test name and timestamp for easy identification
- **Example**: `test_login_20260505_200436_123456.png`

## Running Tests with Report Generation

### Standard Run (All Reports)
```bash
pytest
```

This will automatically generate:
- Allure results
- HTML report
- Logs
- Screenshots (if any tests fail)

### Run Specific Test Suite
```bash
pytest tests/ui/
pytest tests/api/
pytest tests/e2e/
```

### Run with Markers
```bash
pytest -m ui           # Run only UI tests
pytest -m api          # Run only API tests
pytest -m e2e          # Run only E2E tests
```

### Run with More Verbosity
```bash
pytest -vv            # Verbose output
pytest -s             # Show print statements
pytest -vv -s         # Both verbose and print statements
```

## Report Directory Structure

```
reports/
├── allure-results/          # Raw Allure data files
│   ├── *.json              # Test result files
│   ├── *-attachment.txt    # Log attachments
│   └── *-container.json    # Test container info
├── logs/
│   └── pytest.log          # Complete execution logs
├── screenshots/            # Failed test screenshots
│   └── test_name_timestamp.png
└── report.html             # HTML report
```

## Configuration

Report generation is configured in `pytest.ini`:

```ini
addopts =
    --alluredir=./reports/allure-results
    --clean-alluredir
    --html=./reports/report.html
    --self-contained-html
    --log-file=./reports/logs/pytest.log
    --log-level=INFO
    -v
```

## Tips for Using Reports

1. **Quick Check**: Open `report.html` for a quick overview
2. **Detailed Analysis**: Use Allure reports for interactive analysis
3. **Debugging**: Check screenshots and logs for failed tests
4. **CI/CD Integration**: Allure reports can be integrated with CI/CD pipelines
5. **Trend Analysis**: Run tests regularly and compare reports over time

## Troubleshooting

- **Screenshots not appearing**: Check if tests actually failed (screenshots only appear on failures)
- **Allure command not found**: Install with `npm install -g allure-commandline`
- **Report file size too large**: HTML reports can be large with many tests; consider using Allure for better performance
